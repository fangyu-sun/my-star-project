# 项目开发与工作规范

为确保项目开发过程的高效性、代码可追溯性以及版本管理的规范性，特制定以下开发规范：

## Git 提交与模块开发规范

1. **模块化自动提交**：
   * 每当**完成一个功能模块**且**测试通过（编译成功且功能运行正常）**后，自动执行 `git add` 和 `git commit`。
   * 严禁将多个不同功能模块的大量变更堆积到单次提交中，以确保每次提交的原子性。

2. **提交信息（Commit Message）要求**：
   * 提交信息必须简要、清晰地描述本次提交的具体内容和修改范围。
   * 格式规范示例：
     * `feat(satellite): 引入 satellite.js 精确推算人造卫星天顶飞越轨迹`
     * `fix(astronomy): 修复天体筛选圆锥角度至严格的 80 度天顶范围`
     * `docs(workflow): 新增项目 Git 提交与自动模块化开发工作规范`
